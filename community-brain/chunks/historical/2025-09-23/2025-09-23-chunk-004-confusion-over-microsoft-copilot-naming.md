# Confusion over Microsoft Copilot naming

**Session:** Weekly Coaching Call | **Date:** 2025-09-23

**Summary:** Marc and Paul discussed the utility of using Microsoft Copilot to create restricted, RAG-based agents for internal documentation like company policies. They concluded that while this specific function offers practical, secure use cases for individual users, it lacks the broader, "full-blown" capabilities anticipated by some stakeholders.

[00:07:41] Marc Juretus: Yeah, mean, the one cool thing it does do, though, is like I said, I can see a usage where, with the regular Co-Pilot, where you can create an agent and have like a rag against like a 20-page document, and say, okay, for my area, this is how we handle PTO, downtime, disaster recovery, ask the agent.
[00:08:00] Marc Juretus: You can assign it to a specific user's ID and only they have access to it.
[00:08:04] Marc Juretus: So I could see a very good use case for that part of Microsoft Copilot, but it's not the full blown one that I think they think it is.
[00:08:13] Paul Miller: Yeah, that's a good point because within a corporate environment, you kind of don't want them.
